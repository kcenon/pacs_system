# Migration Guide: 0.x to 1.0

This guide walks every breaking change introduced between the `0.1.0` baseline
(2026-03-13) and the `1.0.0` API freeze (2026-05-10). Each section lists the
symbol or surface that moved, the rationale, and a before/after snippet so a
downstream consumer can mechanically update their code.

> **Stability promise**: After v1.0 ships, the public API surface enumerated in
> [`docs/v1.0-api-surface.md`](../v1.0-api-surface.md) is frozen until the v2.0
> SemVer bump. No removal, rename, or signature break will happen on v1.x.
> See the parent EPIC [#1095](https://github.com/kcenon/pacs_system/issues/1095)
> and the ecosystem contract in
> [common_system #639](https://github.com/kcenon/common_system/issues/639).

## Overview

v1.0 is the consolidation release that locks the public API contract for the
foreseeable future. Pre-1.0 minor versions accumulated a backlog of breaking
changes — header-path renames, namespace migrations, weak-crypto removals,
DICOM-record cleanups, and a uniform `Result<T>` error-handling contract.

This guide is split into two parts:

1. **Breaking changes**: every removed, renamed, or signature-broken symbol with
   a paired before/after snippet. Apply these in any order; they are
   independent.
2. **`Result<T>` migration end-to-end**: a longer narrative for consumers whose
   code relied on exceptions escaping the public API. This explains the v1.0
   error-as-value contract, the small set of permitted ABI escapes, and how to
   refactor a typical call site.

The breaking changes below are tracked against specific PRs and issues; the
table at the bottom of each section names the canonical reference for further
investigation.

| Change category | Count |
|-----------------|------:|
| Path / namespace | 2 |
| Removed classes / fields | 4 |
| Behavioural / semantic | 4 |

Read [the Upgrade checklist](#upgrade-checklist) once you have applied the
relevant sections; it acts as a self-audit before you raise a v1.0 PR
downstream.

## Breaking changes

### 1. Include-path migration: `pacs/` → `kcenon/pacs/`, `.hpp` → `.h`

**What changed.** Every public header moved from `include/pacs/...` to
`include/kcenon/pacs/...`, and every `.hpp` extension became `.h`. The change
touched 299 files renamed and ~2,789 include directives updated across 774
source files. The migration aligns pacs_system with the ecosystem-wide
`<kcenon/<project>/...>` include convention and the `.h` extension already used
by the rest of the kcenon stack (`common_system`, `network_system`,
`thread_system`, `logger_system`).

**Reference.** PR [#1055](https://github.com/kcenon/pacs_system/pull/1055),
closes [#1053](https://github.com/kcenon/pacs_system/issues/1053).

**Before (0.x).**

```cpp
#include <pacs/core/dicom_dataset.hpp>
#include <pacs/network/association.hpp>
#include <pacs/services/cstore_scp.hpp>
```

**After (1.0).**

```cpp
#include <kcenon/pacs/core/dicom_dataset.h>
#include <kcenon/pacs/network/association.h>
#include <kcenon/pacs/services/cstore_scp.h>
```

**Mechanical fix.** A repository-wide regex usually suffices:

```bash
# Replace include prefix
grep -rl '#include <pacs/' your_project/ \
  | xargs sed -i 's|#include <pacs/|#include <kcenon/pacs/|g'

# Rename .hpp → .h in include directives only (do not rename your own files)
grep -rl 'kcenon/pacs/.*\.hpp' your_project/ \
  | xargs sed -i 's|\(kcenon/pacs/[^>"]*\)\.hpp|\1.h|g'
```

### 2. Namespace migration: `pacs::` → `kcenon::pacs::`

**What changed.** The canonical namespace is now `kcenon::pacs::`. The legacy
`pacs::` namespace is kept alive as a namespace alias through the v1.0 surface
header [`compat/namespace_compat.h`](../../include/kcenon/pacs/compat/namespace_compat.h),
so existing code that has not been migrated still compiles. New code should use
the canonical spelling directly.

**Reference.** PR [#1055](https://github.com/kcenon/pacs_system/pull/1055)
(rename), confirmation [#1138](https://github.com/kcenon/pacs_system/issues/1138).
Header [`compat/namespace_compat.h`](../../include/kcenon/pacs/compat/namespace_compat.h)
is itself part of the frozen v1.0 surface; it stays until v2.0.

**Before (0.x).**

```cpp
#include <pacs/core/dicom_tag.hpp>

void register_tag(const pacs::core::dicom_tag& tag) {
    // ...
}
```

**After (1.0) — preferred.**

```cpp
#include <kcenon/pacs/core/dicom_tag.h>

void register_tag(const kcenon::pacs::core::dicom_tag& tag) {
    // ...
}
```

**After (1.0) — compatibility shim (existing code base, gradual migration).**

```cpp
#include <kcenon/pacs/core/dicom_tag.h>
#include <kcenon/pacs/compat/namespace_compat.h>  // enables pacs:: alias

// Legacy code continues to compile because pacs = kcenon::pacs.
void register_tag(const pacs::core::dicom_tag& tag) {
    // ...
}
```

### 3. `thread_adapter` class removal

**What changed.** The standalone `pacs::integration::thread_adapter` class and
its companion test file were deleted. Shared types previously declared in
`thread_adapter.hpp` were moved to
[`include/kcenon/pacs/integration/thread_pool_interface.h`](../../include/kcenon/pacs/integration/thread_pool_interface.h),
and six integration tests were rewritten to use the dependency-injection
pattern through `thread_pool_adapter`.

**Reference.** PR [#1000](https://github.com/kcenon/pacs_system/pull/1000).

**Before (0.x).**

```cpp
#include <pacs/integration/thread_adapter.hpp>

auto adapter = std::make_shared<pacs::integration::thread_adapter>();
adapter->configure(...);
my_service.attach_executor(adapter);
```

**After (1.0).**

```cpp
#include <kcenon/pacs/integration/thread_pool_adapter.h>
#include <kcenon/pacs/integration/thread_pool_interface.h>  // shared types

auto pool    = std::make_shared<kcenon::thread::thread_pool>(/* workers */ 8);
auto adapter = std::make_shared<kcenon::pacs::integration::thread_pool_adapter>(pool);
my_service.attach_executor(adapter);
```

The adapter is now constructed with an injected `thread_pool` rather than
configuring an embedded pool internally. This makes lifetime ownership explicit
and removes the singleton-flavoured `configure()` step.

### 4. `value_conversion_error` exception removal

**What changed.** The exception class
`pacs::core::value_conversion_error` was removed from
`include/pacs/core/dicom_element.hpp` (now
`include/kcenon/pacs/core/dicom_element.h`). All call sites that previously
threw this exception now return `Result<T>` and surface
`error_codes::value_conversion_error` (`-721`) on failure.

**Reference.** PR [#1000](https://github.com/kcenon/pacs_system/pull/1000).
Error code defined in
[`include/kcenon/pacs/core/result.h`](../../include/kcenon/pacs/core/result.h).

**Before (0.x).**

```cpp
try {
    int patient_age = element.get_value<int>();
    log("age = {}", patient_age);
} catch (const pacs::core::value_conversion_error& e) {
    log_error("conversion failed: {}", e.what());
}
```

**After (1.0).**

```cpp
auto age_result = element.get_value<int>();
if (kcenon::common::is_error(age_result)) {
    const auto& err = kcenon::common::get_error(age_result);
    log_error("conversion failed: code={} message={}", err.code, err.message);
    return;
}
int patient_age = kcenon::common::get_value(age_result);
log("age = {}", patient_age);
```

The general `Result<T>` migration recipe applies — see
[Result-T migration end-to-end](#result-t-migration-end-to-end).

### 5. Weak-crypto MAC algorithm removal

**What changed.** The `sha1`, `md5`, and `ripemd160` enumerators were removed
from `mac_algorithm` and from every switch/case handler that consumed them.
These algorithms are cryptographically broken or weakened and were retained
only for transitional compatibility during 0.x.

**Reference.** PR [#1000](https://github.com/kcenon/pacs_system/pull/1000).

**Before (0.x).**

```cpp
using namespace pacs::security;

auto cfg = mac_config{
    .algorithm = mac_algorithm::sha1,   // removed in 1.0
    .key       = my_key,
};
auto mac = compute_mac(cfg, payload);
```

**After (1.0).**

```cpp
#include <kcenon/pacs/security/mac.h>

using namespace kcenon::pacs::security;

auto cfg = mac_config{
    .algorithm = mac_algorithm::sha256,  // or sha384 / sha512
    .key       = my_key,
};
auto mac_result = compute_mac(cfg, payload);
```

If your downstream code currently relies on a removed enumerator, choose the
nearest SHA-2 (or SHA-3) replacement. A compile error is the intended UX — the
weak algorithms are not silently re-aliased.

### 6. `body_part_thickness_mm` field removal

**What changed.** The field `body_part_thickness_mm` was removed from
`mg_acquisition_params` (mammography acquisition parameters) and replaced by
`compressed_breast_thickness_mm`. The replacement field name matches DICOM
attribute `(0018,11A0) Body Part Thickness` semantics more precisely; the old
name caused integrators to mistakenly populate uncompressed measurements.

**Reference.** PR [#1000](https://github.com/kcenon/pacs_system/pull/1000).

**Before (0.x).**

```cpp
pacs::services::mg::mg_acquisition_params params;
params.body_part_thickness_mm = 47.5;   // removed in 1.0
```

**After (1.0).**

```cpp
kcenon::pacs::services::mg::mg_acquisition_params params;
params.compressed_breast_thickness_mm = 47.5;
```

Field rename only; the type (`double`) and semantics (millimetres) are
unchanged.

### 7. `database_system` API migration

**What changed.** Pacs storage migrated to the current `database_system`
public API:

- `query_condition(string)` → `query_condition(field, op, value)` /
  `builder.where(field, op, value)`.
- The split DML result types
  (`insert_query_result`, `update_query_result`, `delete_query_result`) were
  unified into a single `execute_query_result`.
- `build_dicom_condition(...)` (which returned raw SQL) was refactored into
  `add_dicom_condition(builder, ...)` which mutates the builder directly.

**Reference.** PR [#1003](https://github.com/kcenon/pacs_system/pull/1003).
Affected internal files: `src/storage/audit_repository.cpp`,
`src/storage/mpps_repository.cpp`, `src/storage/sqlite_security_storage.cpp`,
`src/storage/ups_repository.cpp`, `src/storage/worklist_repository.cpp`. If
your downstream code constructs `pacs_system` storage repositories or composes
custom queries against the same builder, mirror the patterns below.

**Before (0.x).**

```cpp
auto cond  = query_condition("study_date >= '2026-01-01'");
auto rows  = repo.find_by(cond);

auto raw_sql = build_dicom_condition(tag::patient_id, value);
auto rows2   = repo.execute_raw(raw_sql);

insert_query_result ins = repo.insert(record);
update_query_result upd = repo.update(record);
delete_query_result del = repo.remove(id);
```

**After (1.0).**

```cpp
auto cond  = query_condition("study_date", op::gte, "2026-01-01");
auto rows  = repo.find_by(cond);

query_builder qb;
add_dicom_condition(qb, tag::patient_id, value);
auto rows2 = repo.execute(qb);

execute_query_result ins = repo.insert(record);
execute_query_result upd = repo.update(record);
execute_query_result del = repo.remove(id);
```

For full upstream context, see
[`docs/database/MIGRATION_GUIDE.md`](../database/MIGRATION_GUIDE.md).

### 8. `patient_age` removal from basic anonymization profile

**What changed.** The basic anonymization profile now removes the
`(0010,1010) Patient's Age` attribute in addition to its prior set, to comply
with HIPAA Safe Harbor requirements. Downstream code that asserted on the
presence of `PatientAge` in the basic profile output must be updated.

**Reference.** PR [#999](https://github.com/kcenon/pacs_system/pull/999) and
PR [#993](https://github.com/kcenon/pacs_system/pull/993).

**Before (0.x).**

```cpp
auto anon = basic_profile.apply(original);
EXPECT_TRUE(anon.has_tag(tag::patient_age));
```

**After (1.0).**

```cpp
auto anon = basic_profile.apply(original);
EXPECT_FALSE(anon.has_tag(tag::patient_age));  // removed by Safe Harbor
```

If your application still needs `PatientAge` for downstream analytics, opt into
a **clinical-trial profile** (or a custom profile) instead of the basic
profile, and document the regulatory basis.

### 9. Directory layout: `samples/` ↔ `examples/` ↔ `tools/`

**What changed.** Two directories swapped roles and one moved to align with the
ecosystem convention:

- The five progressive tutorials previously under `samples/` are now under
  `examples/`.
- The 32 CLI utility binaries previously under `examples/` are now under
  `tools/`.

The CMake option names (`PACS_BUILD_EXAMPLES`, `PACS_BUILD_SAMPLES`) are
preserved for backward compatibility — `PACS_BUILD_EXAMPLES` continues to
control the tutorial build, and `PACS_BUILD_SAMPLES` is retained as an alias.
**Path references in downstream build files must be updated.**

**Reference.** PR [#1139](https://github.com/kcenon/pacs_system/pull/1139) and
the BREAKING entry in the [Unreleased] CHANGELOG section.

**Before (0.x).**

```cmake
# Tutorials
add_subdirectory(${pacs_SOURCE_DIR}/samples/getting_started)

# CLI utilities
target_include_directories(my_tool PRIVATE ${pacs_SOURCE_DIR}/examples/include)

# Binary output
file(COPY ${CMAKE_BINARY_DIR}/samples/foo.dat DESTINATION ${output_dir})
```

**After (1.0).**

```cmake
# Tutorials
add_subdirectory(${pacs_SOURCE_DIR}/examples/getting_started)

# CLI utilities
target_include_directories(my_tool PRIVATE ${pacs_SOURCE_DIR}/tools/include)

# Binary output
file(COPY ${CMAKE_BINARY_DIR}/examples/foo.dat DESTINATION ${output_dir})
```

### 10. `.key` certificate removal from version control

**What changed.** Committed `*.key` private-key files are gone from the
repository. They are regenerated on-the-fly by `generate_test_certs.sh` during
CI and local development.

**Reference.** PR [#1092](https://github.com/kcenon/pacs_system/pull/1092).

**Before (0.x) — downstream test setup.**

```bash
# Tests assumed pacs_system shipped a stable key file
cp third_party/pacs_system/tests/data/server.key ./fixtures/server.key
```

**After (1.0).**

```bash
# Generate keys per build / CI run
third_party/pacs_system/scripts/generate_test_certs.sh ./fixtures/
```

If your CI cached the old `.key` files, invalidate the cache and rely on the
generator script. Do not re-commit private keys.

## Result-T migration end-to-end

The v1.0 public API contract is **error-as-value via `Result<T>`**. This
section walks the migration in detail for consumers whose code relied on
exceptions escaping the public surface.

### The new contract in one paragraph

Public headers under `include/kcenon/pacs/` contain zero non-comment `throw`
statements. Recoverable failures from any public entry point are returned as
`kcenon::common::Result<T>` (alias `kcenon::pacs::Result<T>`). A small inventory
of permitted ABI escapes is documented in
[`docs/v1.0-throw-policy.md`](../v1.0-throw-policy.md); these are constructor
invariants and `std::future`-boundary re-throws, never normal call paths.

### Step 1 — Recognise the alias surface

```cpp
// include/kcenon/pacs/core/result.h
namespace kcenon::pacs {
    template <typename T>
    using Result = kcenon::common::Result<T>;

    using VoidResult = kcenon::common::VoidResult;
    using error_info = kcenon::common::error_info;

    namespace error_codes { /* common + pacs-specific codes */ }
}
```

A typical signature you will see in the v1.0 surface:

```cpp
[[nodiscard]] Result<dicom_dataset>
read_dicom_file(const std::filesystem::path& path);
```

The `[[nodiscard]]` attribute is intentional. Ignoring the `Result<T>` is a
compile-time warning and a `-Werror` failure in CI-grade builds.

### Step 2 — Replace `try / catch` with `is_error` / `get_value`

**Before (0.x).**

```cpp
try {
    auto ds = pacs::core::read_dicom_file(path);
    process(ds);
} catch (const pacs::core::value_conversion_error& e) {
    handle_conversion_error(e.what());
} catch (const std::runtime_error& e) {
    handle_runtime_error(e.what());
}
```

**After (1.0).**

```cpp
auto result = kcenon::pacs::core::read_dicom_file(path);
if (kcenon::common::is_error(result)) {
    const auto& err = kcenon::common::get_error(result);
    switch (err.code) {
        case kcenon::pacs::error_codes::value_conversion_error:
            handle_conversion_error(err.message);
            break;
        case kcenon::pacs::error_codes::file_read_error:
            handle_runtime_error(err.message);
            break;
        default:
            handle_runtime_error(err.message);
            break;
    }
    return;
}
process(kcenon::common::get_value(result));
```

### Step 3 — Chain calls with the monadic helpers

For pipelines that previously stacked multiple try/catch layers, prefer the
monadic combinators provided by `kcenon::common::Result`:

```cpp
auto pipeline_result =
    read_dicom_file(path)
        .and_then([](auto ds)   { return validate_dataset(ds); })
        .and_then([](auto ds)   { return store_dataset(ds); })
        .map     ([](auto info) { return info.stored_uid; });

if (kcenon::common::is_error(pipeline_result)) {
    log_error(kcenon::common::get_error(pipeline_result));
    return;
}
auto stored_uid = kcenon::common::get_value(pipeline_result);
```

`and_then` short-circuits on the first error and propagates the `error_info`
unchanged, so a single error-handling block at the end of the chain replaces a
nested try/catch ladder.

### Step 4 — Treat constructor and `std::future` throws as documented escapes

The following call paths still throw in v1.0. They are documented exhaustively
in [`docs/v1.0-throw-policy.md`](../v1.0-throw-policy.md):

- `thread_pool_adapter(std::shared_ptr<thread_pool>)` — null-pointer invariant.
- `thread_pool_executor_adapter` constructor variants — null-pointer and
  start-failure invariants.
- `make_executor()` factory — null-pointer invariant.
- `hsm_storage(...)` constructor — null-pointer invariant on the hot tier.
- Lambdas submitted to a `std::future`-returning `submit()` — re-throw at
  `future::get()`, idiomatic to `std::future`.

When wiring these types into your own code, **either** wrap construction in a
`try / catch` for the invariant errors, **or** validate inputs before
construction (the preferred pattern):

```cpp
// Preferred — validate at the boundary
if (!hot_tier) {
    log_error("hot_tier is required");
    return;
}
auto storage = std::make_shared<kcenon::pacs::storage::hsm_storage>(
    hot_tier, warm_tier, cold_tier);
```

### Step 5 — Migrate your own helpers to the same contract

If you have intermediate helpers that wrap pacs_system APIs, replace their
`throw`-based signatures with `Result<T>` returns to keep the contract
transitive:

**Before (0.x).**

```cpp
dicom_dataset load_or_die(const std::filesystem::path& p) {
    auto ds = pacs::core::read_dicom_file(p);  // threw on error
    if (!ds.has_required_tags()) {
        throw std::runtime_error("missing required tags");
    }
    return ds;
}
```

**After (1.0).**

```cpp
Result<dicom_dataset>
load_or_report(const std::filesystem::path& p) {
    auto ds_result = kcenon::pacs::core::read_dicom_file(p);
    if (kcenon::common::is_error(ds_result)) {
        return ds_result;  // forward the error
    }
    auto ds = kcenon::common::get_value(ds_result);
    if (!ds.has_required_tags()) {
        return kcenon::common::error<dicom_dataset>(
            kcenon::pacs::error_codes::element_not_found,
            "missing required tags");
    }
    return kcenon::common::ok(std::move(ds));
}
```

This keeps every layer between user code and pacs_system honest about
failures: there are no hidden control-flow exits.

## Upgrade checklist

Run through this list before raising a v1.0 upgrade PR downstream.

- [ ] All `#include <pacs/...>` rewritten to `#include <kcenon/pacs/...>`.
- [ ] All `.hpp` suffixes in pacs-system include directives replaced with
      `.h`.
- [ ] All `pacs::` qualifiers either rewritten to `kcenon::pacs::` or backed
      by `#include <kcenon/pacs/compat/namespace_compat.h>` during a
      gradual migration.
- [ ] No remaining references to `thread_adapter`; executor wiring uses
      `thread_pool_adapter` with an injected `thread_pool`.
- [ ] No remaining references to `pacs::core::value_conversion_error`;
      conversion failures are handled through `Result<T>` and
      `error_codes::value_conversion_error`.
- [ ] No remaining references to `mac_algorithm::sha1`, `::md5`, or
      `::ripemd160`. Weakest replacement is `sha256`.
- [ ] No remaining writes to `body_part_thickness_mm`; the field name is
      `compressed_breast_thickness_mm`.
- [ ] Storage call sites use the unified `execute_query_result` DML return
      type, 3-arg `query_condition`, and `add_dicom_condition(qb, ...)`.
- [ ] Anonymization-profile tests do not assert that
      `tag::patient_age` survives the basic profile.
- [ ] Build files reference `examples/` for tutorials and `tools/` for the
      CLI utilities (not the swapped 0.x paths).
- [ ] CI no longer caches `*.key` files from the pacs_system source tree;
      keys are produced by `scripts/generate_test_certs.sh`.
- [ ] All `try / catch` around pacs_system public API calls replaced with
      `is_error` / `get_value` (or `and_then` / `map` chains).
- [ ] Documented ABI escapes from
      [`docs/v1.0-throw-policy.md`](../v1.0-throw-policy.md) are still
      wrapped where they apply (constructors and `std::future` boundaries).
- [ ] All intermediate helpers in your own code return `Result<T>` instead of
      throwing, so the error contract stays transitive end to end.

When every box is ticked, your downstream project is on the v1.0 contract and
will benefit from the v1.0–v2.0 stability promise.
