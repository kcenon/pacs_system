# v1.0 Throw Policy and ABI Escape Inventory

> **Issue**: [#1157](https://github.com/kcenon/pacs_system/issues/1157)
> **Part of**: [#1095](https://github.com/kcenon/pacs_system/issues/1095) — v1.0 API freeze
> **Last updated**: 2026-05-10

## Policy

The v1.0 PACS public API contract is **error-as-value via `Result<T>`**:

- Public headers under `include/kcenon/pacs/` MUST NOT contain `throw` statements
  in any non-comment line.
- Implementation files (`src/`) backing public APIs SHOULD return
  `kcenon::common::Result<T>` for all recoverable failure modes.
- A small set of throws is permitted as **documented ABI escapes** when the
  affected entry point is a constructor (no Result return path possible) or
  when the throw propagates strictly through a `std::future` boundary that
  consumers already drain via `future::get()` (idiomatic re-throw).

This file is the canonical inventory of those escapes for v1.0. New throws
outside this inventory are a freeze violation.

## Header Audit (v1.0 baseline)

`grep -rn 'throw' include/` returns 14 matches; all are documentation strings
(`@throws` Doxygen tags or prose like "never throws"). **Zero non-comment
`throw` statements** exist in public headers.

## Source Audit (v1.0 baseline)

`grep -rEn '^[[:space:]]*throw[[:space:]]' src/` returns 12 matches across 4
files. Each is classified below.

### Net Result

- **Throws migrated**: 2 (true Result&lt;T&gt; bypasses fixed)
- **Throws remaining as documented ABI escapes**: 10
- **Throws in public headers**: 0

## Maintenance Rules

1. Any PR that adds a `throw` to `src/**` MUST either:
   - Migrate the call path to return `Result<T>`, or
   - Add an entry to the "Permitted ABI Escapes" table above with a
     classified rationale, or
   - Be rejected.
2. Any PR that adds a non-comment `throw` to `include/kcenon/pacs/**` MUST be rejected.
3. CI may enforce the header rule with `grep -rEn '^[[:space:]]*throw[[:space:]]' include/` returning zero matches.
