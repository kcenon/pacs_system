OAuth 2.0 middleware for DICOMweb endpoint authorization.

OAuth 2.0 middleware for DICOMweb endpoint authorizationWhen OAuth 2.0 is enabled, the middleware:

  1. Extracts the Bearer token from the Authorization header
  2. Decodes and validates the JWT (issuer, audience, expiration)
  3. Verifies the token signature using JWKS keys
  4. Checks required scopes for the DICOMweb operation
  5. Creates a user_context for the downstream endpoint

When OAuth 2.0 is disabled, the middleware is a no-op and the system falls back to the existing X-User-ID header-based authentication.

oauth2_config config;
config.enabled = true;
config.issuer = "https://auth.example.com";
oauth2_middleware middleware(config);
middleware.set_jwks_provider(jwks);
// In endpoint handler:
auto result = middleware.authenticate(req, res);
if (!result) return res; // Error response already set
if (!middleware.require_scope(result->claims, res, dicomweb_scopes::read)) {
return res; // 403 Forbidden
}
